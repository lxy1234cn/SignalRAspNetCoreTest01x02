# 🚀 SignalR 多分区聊天系统

一个基于 **ASP.NET Core + SignalR** 构建的实时聊天实验项目。

支持：

-   ✅ 公共广播聊天
-   ✅ 私聊（点对点）
-   ✅ 组聊天（多房间）
-   ✅ 自动重连
-   ✅ 会话分区管理
-   ✅ 未读消息提示
-   ✅ 响应式前端界面

------------------------------------------------------------------------

# 📦 技术栈

## 后端

-   .NET 8
-   ASP.NET Core Web API
-   SignalR
-   Swagger（已封装 JWT 扩展）
-   CORS 全开放策略（实验用途）

## 前端

-   原生 HTML + CSS + JavaScript
-   Microsoft SignalR JS 8.0.0
-   响应式设计（支持移动端）

# 🔥 核心功能说明

## 1️⃣ 连接管理

### ConnectionManager（单例）

使用：

``` csharp
ConcurrentDictionary<string,string>
```

功能：

-   记录用户连接
-   根据 userId 获取 connectionId
-   自动移除断开的连接

注册方式：

``` csharp
builder.Services.AddSingleton<ConnectionManager>();
```

------------------------------------------------------------------------

## 2️⃣ ChatHub 功能

### 连接生命周期

#### OnConnectedAsync

-   从 QueryString 获取 `userId`
-   绑定 connectionId
-   向客户端返回 ConnectionId（ConnectionAck）

#### OnDisconnectedAsync

-   自动移除连接记录

------------------------------------------------------------------------

### 广播消息

``` csharp
public Task BroadcastMessage(MessageDto msg)
```

发送给所有在线客户端。

------------------------------------------------------------------------

### 私聊消息

``` csharp
public Task SendMessageToUser(string targetConnectionId, MessageDto msg)
```

点对点发送。

------------------------------------------------------------------------

### 组消息

``` csharp
JoinGroup(string groupName)
LeaveGroup(string groupName)
SendMessageToGroup(string groupName, MessageDto msg)
```

支持：

-   动态创建组
-   加入组
-   退出组
-   向组广播

------------------------------------------------------------------------

## 3️⃣ 时间戳机制

所有消息都会自动确保拥有 ISO 8601 时间戳：

``` csharp
DateTime.UtcNow.ToString("O")
```

# 🖥 前端机制说明

## 会话分区管理

前端维护：

``` javascript
chatSections = {
  public: [],
  groupA: [],
  pm_xxx: []
}
```

支持：

-   公共频道
-   动态组频道
-   动态私聊频道

------------------------------------------------------------------------

## 自动重连机制

``` javascript
.withAutomaticReconnect()
```

支持：

-   断线自动重连
-   重连成功后自动加入原有组

------------------------------------------------------------------------

## 未读消息提示

-   非当前会话收到消息
-   自动增加红点数量
-   切换会话自动清除

------------------------------------------------------------------------

## 点击头像私聊

-   点击头像
-   自动创建私聊分区
-   切换到私聊窗口

# 🧠 系统设计特点

## 前端负责分区逻辑

Hub 只负责：

-   消息转发
-   连接管理

前端负责：

-   会话分区
-   UI 管理
-   未读逻辑

优点：

-   解耦
-   易扩展
-   易实验
-   逻辑清晰
